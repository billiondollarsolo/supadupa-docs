# Organizations

Organizations group projects and access. Create an organization before creating projects unless the environment already has a seeded default organization.

## Typical Tasks

- Create an organization.
- Invite or add members.
- Assign teams.
- Review project access.
- Inspect quota and usage.

## First Setup

1. Open `Organizations`.
2. Create `Demo Org` or the real organization name.
3. Add initial operators.
4. Confirm access review state before creating production-like projects.
